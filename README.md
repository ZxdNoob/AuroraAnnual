# AuroraAnnual ✨

## 项目简介

AuroraAnnual 是一个充满仪式感的年度项目集合。每当跨年之际，我们都会在这里新增一个子项目，寓意着每年都会迎来好运，接收到满满吉祥气运。这不仅是对新一年的美好祝愿，更是记录成长与收获的时光印记。

## 项目特色

- **跨年仪式感**：本项目每遇到跨年（当年的 12 月 31 日到下一年的元旦）都会新增一个子项目
- **美好寓意**：象征着新的一年的开始，承载着对未来的美好祝愿和期待
- **成长记录**：随着每年的跨年，项目会不断添加新的子项目，记录每一年的成长与收获
- **持续更新**：跨年项目不意味着只在元旦前一天和元旦当天更新，日后也可以持续更新和完善。只是说每遇到跨年时新增一个子项目，每个子项目都可以在创建后的任何时间进行更新和维护

## 项目结构

随着每年的跨年，项目会不断添加新的子项目，每个子项目都代表着新一年的开始和新的希望。

### 子项目命名规范

- **命名格式**：`thisYearToNextYear`（如 `2025To2026`）
- **命名说明**：表示从 thisYear 年底跨越到 nextYear 年初的项目
- **示例**：
  - `2025To2026`：表示 2025 年底跨越到 2026 年初的项目
  - `2026To2027`：表示 2026 年底跨越到 2027 年初的项目

### 当前子项目

#### 2025To2026 - 全栈学习激励平台

**项目简介**：全栈且兼容 PC 端和移动端的全栈学习 + 笔记 + 练习 + 打卡 + 积分 + 段位 + 勋章 + 抽奖激励机制平台。

**核心功能**：

- 用户系统（唯一超级管理员）
- 在线编码系统（支持新建各类文件）
- 积分系统（连续打卡积分加成，最多 10 分）
- 段位系统（仿照王者荣耀和英雄联盟，13 个段位等级）
- 经验等级系统（登录和打卡获得经验，连续有加成）
- 权益系统（不同等级和段位享有不同权益）
- 抽奖系统（红包、道具、限定奖励）
- 勋章系统（各种成就勋章）

**技术栈**：

- 前端：Next.js 14 + React 19.2 + TypeScript + Tailwind CSS
- 后端：NestJS + TypeScript + Prisma
- 数据库：PostgreSQL + Redis
- 基础设施：Docker + GitHub Actions

**项目目录**：[`2025To2026/`](./2025To2026/)

## ⚠️ 重要提示：规则阅读要求

**每次更新本项目前，必须阅读相关的规则文件！**

### 为什么需要阅读规则？

- **Cursor 可能无法完全缓存所有规则**：项目有 12 个详细的规则文件，内容较多
- **规则会不断更新**：规则文件会随着项目发展而更新
- **确保理解要求**：只有阅读规则才能确保理解所有要求和约束

### 如何阅读规则？

1. **查看规则索引**：首先阅读 [`.cursor/rules/RULES-INDEX.md`](.cursor/rules/RULES-INDEX.md) 了解所有规则文件
2. **根据操作类型选择**：根据当前操作类型，阅读对应的规则文件
3. **理解关键要求**：重点关注与当前操作相关的规则要求
4. **遵循规则执行**：严格按照规则要求执行操作

### 快速开始

- **规则索引**：[`.cursor/rules/RULES-INDEX.md`](.cursor/rules/RULES-INDEX.md) - 包含所有规则的索引和阅读指南
- **规则目录**：`.cursor/rules/` - 所有规则文件都在此目录
